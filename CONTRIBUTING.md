# Contributing to AI Secret Scanner

First off — thank you for considering a contribution! This is an open-source security research tool and every improvement makes it more effective at finding exposed credentials.

## Adding a New Secret Pattern

Edit `src/scanner/patterns.js`:

```js
// Add to the PATTERNS array:
{
  id: 'my_provider_key',           // unique snake_case id
  name: 'MyProvider API Key',      // human-readable
  provider: 'myprovider',          // lowercase provider name
  regex: /myprovider_[A-Za-z0-9]{32}/,
  group: 0,                        // 0 = full match, 1+ = capture group
  // optional: only match if context contains this regex
  requireContext: /my_provider|MY_PROVIDER/i
}
```

**Rules for patterns:**
1. Must NOT match dummy/test values (the false-positive filter will catch most, but be specific)
2. Minimum length of matched value must be ≥ 16 chars
3. Add `requireContext` for generic-looking patterns to reduce false positives
4. Test your pattern against real examples from the provider's docs (never real leaked keys)

## Adding a New Validator

Edit `src/validator/index.js` — add to the `validators` object:

```js
async myprovider(secret) {
  try {
    const resp = await http.get('https://api.myprovider.com/v1/me', {
      headers: { Authorization: `Bearer ${secret}` }
    });
    return {
      result: resp.status === 200 ? RESULTS.VALID : RESULTS.INVALID,
      detail: `HTTP ${resp.status}`
    };
  } catch (err) {
    if (err.response?.status === 401)
      return { result: RESULTS.INVALID, detail: 'Unauthorized' };
    return { result: RESULTS.ERROR, detail: err.message };
  }
}
```

**Rules for validators:**
1. Only make `GET` requests (never mutating calls)
2. Use the minimum-permission endpoint (e.g. `/me`, `/whoami`, `/models`)
3. Always handle 401 → INVALID and network errors → ERROR
4. Never log the raw secret value — use `finding.value` (redacted) for logs

## Pull Request Process

1. Fork the repo
2. Create a branch: `git checkout -b feat/my-new-provider`
3. Make your changes
4. Test: `node -e "const {PATTERNS}=require('./src/scanner/patterns'); console.log(PATTERNS.length)"`
5. Push and open a PR against `main`
6. Describe what provider/pattern you added and why it's valuable

---

## Code Style

- Node.js (CommonJS `require`)
- `'use strict'` at the top of every file
- No TypeScript required — plain JS is fine
- Camelcase variables, descriptive names
- Add JSDoc comments for public functions

---

## Security Policy

If you discover a vulnerability in this scanner itself (not a finding from a scan), please open a private GitHub Advisory rather than a public issue.

**Do not** include real leaked API keys in PRs, issues, or test files — use redacted placeholders like `sk-proj-****`.
